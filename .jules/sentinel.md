## 2024-05-24 - File Download Path Traversal Vulnerabilities
**Vulnerability:** The application used `send_file` directly with user-influenced path variables (e.g., `temp_path`, `config_path`, `script_path`), creating significant risk for Local File Inclusion/Path Traversal.
**Learning:** In Flask, `send_file` does not provide robust protection against path traversal when the path originates from or is concatenated with user input. Even if paths appear to be constructed safely, it's safer to rely on framework-level protections. Attempting to use `os.path.dirname(path)` to extract a directory and pass it to `send_from_directory` is an anti-pattern (security theater) if the base directory itself isn't a known, trusted constant.
**Prevention:** Always use `send_from_directory` instead of `send_file` for file downloads. The first argument (the directory) MUST be a trusted, fixed, server-defined path (e.g., `sim_dir`, `scripts_dir`, `temp_dir`), and the second argument is the strictly sanitized filename.
