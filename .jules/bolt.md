## 2024-05-08 - Fixing N+1 Queries in Simulation Context Enrichment
**Learning:** Found a specific N+1 query pattern in `backend/scripts/run_parallel_simulation.py` where SQL queries explicitly included a `LEFT JOIN` (e.g., `LEFT JOIN user u ON p.user_id = u.user_id`) to the `user` table, but failed to `SELECT` the necessary columns (`name`, `user_name`). This resulted in a subsequent identical query to the `user` table for every single post and comment processed during context enrichment, completely negating the benefit of the JOIN and creating an N+1 query loop.
**Action:** When performing JOINs, ensure that all needed columns from the joined table are included in the initial `SELECT` clause to avoid redundant secondary queries. Review existing queries that use `LEFT JOIN` to ensure they are actually utilizing the joined table's columns in the SELECT clause.
