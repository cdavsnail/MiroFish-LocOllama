## 2024-05-17 - Eliminate N+1 Queries in Simulation Data Enrichment
**Learning:** In `run_parallel_simulation.py`, the simulation data enrichment functions (`_get_post_info` and `_get_comment_info`) previously used a `LEFT JOIN` on the `user` table but failed to select `u.name` and `u.user_name`. This led to a fallback mechanism where a secondary database query (`cursor.execute(...)`) was executed inside a loop for every row when `agent_names[agent_id]` was unavailable, causing an N+1 query performance bottleneck.
**Action:** When enriching simulation data with related user/author information from SQLite, always include the necessary fields (`u.name`, `u.user_name`) in the `SELECT` clause of the primary SQL `JOIN` query to fetch all required information at once, completely avoiding secondary nested queries inside loops.
